* 'git pull origin master' - will pull down any changes from the GitHub repository. * 'git checkout branch-test' - will switch to named branch. * 'git merge branch-test' - will merge with master, as long as master is the active branch. * To catch up branch-test to the master, 'git checkout branch-test; git merge master' * To move a branch to a specific commit, 'git branch -f branch-test target-commit', eg. 'git branch -f master craft-changes' moves the latest master commit to the current commit of craft-changes.

Heading Attributes

Elements can be assigned attributes using this syntax at the end of the line: {#identifier .class .class key=value key=value} Thus, for example, the following headings will all be assigned the identifier foo: # My heading { #foo}

Filters

  • Convert to uppercase - { page.heading | upcase } //use double brackets
  • Limit number of characters - { page.heading | truncate:8 }
    • //double brackets
  • if/else statements -
    [% if page.show_heading %] //if this parameter in the front matter is true and use {brackets}
    { page.heading | upcase } //double brackets
    [% elsif page.heading contains "cupcake" %] // { & %
    I want cupcakes
    [% else %] // { & %
    I don't want cupcakes
    [% endif %] // { & %
  • Looping -
    [% for cupcake in page.cupcakes %] //list of cupcakes in front matter. // { & %
    ul>
    li>{ cupcake }/li>
    [% endfor %] // { & %
    /ul>
